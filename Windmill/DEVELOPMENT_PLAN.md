# **Windmill 개발 작업 계획**

## **📋 작업 목록 및 우선순위**

### **작업 항목**
1. 간단한 ETL 기능 추가
2. 디비연결 인스턴스 간섭 없도록
3. 파일기반 → DB이전
4. 화면 버그 잡기
5. 전수 테스트 및 버그 수정, 성능개선

---

## **🎯 1단계: 기반 작업 (안정성 확보)**

### **1-1. 화면 버그 잡기** ⭐ **최우선**
**왜 최우선인가?**
- 현재 해시 비교 방식 적용 후 UI 문제 발생 가능성
- 사용자 경험에 직접적 영향
- 후속 작업의 기반이 되는 안정성 확보

**작업 내용:**
- [ ] 대시보드 차트 렌더링 오류 확인 및 수정
- [ ] DEX 상태 카드 레이아웃 문제 해결
- [ ] 해시 비교 로직으로 인한 차트 업데이트 문제 수정
- [ ] 브라우저 호환성 검증 (Chrome, Firefox, Safari)
- [ ] 반응형 디자인 문제 해결
- [ ] 색상 테마 통일성 검증

**예상 소요 시간:** 2-3일

### **1-2. 디비연결 인스턴스 간섭 없도록** ⭐ **중요**
**왜 중요한가?**
- 다중 사용자 환경에서 데이터 충돌 방지
- 시스템 안정성 확보
- 확장성 기반 마련

**작업 내용:**
- [ ] Connection Pool 설정 최적화
- [ ] 세션별 연결 관리 구현
- [ ] 동시 요청 처리 개선
- [ ] 데이터베이스 연결 격리 메커니즘
- [ ] 연결 상태 모니터링 강화
- [ ] 연결 실패 시 재시도 로직

**예상 소요 시간:** 3-4일

## **⚠️ 주의사항 및 리스크 관리**

### **1. 의존성 고려**
- **화면 버그 수정** → **DB 연결 최적화** → **파일→DB 이전** 순서 필수
- ETL 기능은 DB 이전 완료 후 개발
- 각 단계별 완료 후 다음 단계 진행

### **2. 리스크 관리**
- 각 단계별 백업 생성
- 점진적 배포로 위험 최소화
- 롤백 계획 수립
- 코드 버전 관리 철저히

### **3. 테스트 전략**
- 각 단계별 단위 테스트
- 통합 테스트 환경 구축
- 성능 테스트 자동화
- 사용자 수용 테스트
