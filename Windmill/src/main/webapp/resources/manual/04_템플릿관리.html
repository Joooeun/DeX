4. 템플릿 관리

4.1 화면 구성

템플릿 관리 화면은 SQL 템플릿과 카테고리를 관리하는 화면입니다. 템플릿을 카테고리별로 분류하여 관리할 수 있습니다.

템플릿 관리 화면 스크린샷

4.2 기능 설명

1. 카테고리 생성

  • 새 카테고리 버튼 클릭
  • 카테고리명 입력
  • 생성 버튼 클릭

2. 카테고리 수정

  • 카테고리의 수정 버튼 클릭
  • 카테고리명 변경
  • 수정 버튼 클릭

3. 카테고리 삭제

  • 카테고리의 삭제 버튼 클릭
  • 확인 메시지 후 삭제
  • 카테고리에 속한 템플릿은 미분류로 이동

4. 카테고리 순서 변경

  • 위/아래 버튼으로 카테고리 표시 순서 변경
  • 순서는 즉시 저장됩니다

5. 템플릿 목록

선택한 카테고리에 속한 템플릿 목록이 표시됩니다.

  • 템플릿 타입 배지: 템플릿 타입을 색상별 배지로 표시합니다.
    • SQL: 파란색 배지
    • HTML: 주황색 배지
    • SHELL: 초록색 배지
  • 템플릿명: 템플릿의 이름이 표시됩니다.
  • 상태 표시: 비활성 상태인 템플릿은 회색 배경과 취소선으로 표시됩니다.
  • 템플릿 선택: 템플릿을 클릭하면 우측 패널에 템플릿 정보가 로드되어 편집할 수 있습니다.
  • 수정/삭제 버튼: 각 템플릿 항목에 수정 및 삭제 버튼이 있습니다.

6. 템플릿 생성

  • 새 템플릿 버튼 클릭
  • 우측 패널에서 템플릿 정보 입력
  • 저장 버튼 클릭

7. 템플릿 편집

  • 템플릿 목록에서 템플릿 클릭
  • 우측 패널에 템플릿 정보 로드
  • 정보 수정 후 저장

템플릿 편집 상세

7.1 템플릿 기본 정보 설정

  • 템플릿명: 템플릿의 이름
  • 설명: 템플릿에 대한 설명
  • 타입: SQL, HTML, SHELL, PYTHON 중 선택
  • 상태: 활성/비활성
  • 실행 제한: SQL 실행 결과 행 수 제한
  • 새로고침 타임아웃: 자동 새로고침 간격 (초)

7.2 파라미터 관리

템플릿에서 사용할 파라미터를 설정할 수 있습니다. SQL 문에서 \${파라미터명} 형태로 사용되며, 실행 시 실제 값으로 치환됩니다.

  • 순서: 파라미터의 입력 순서를 설정합니다. 숫자가 작을수록 먼저 입력받으며, 사용자 입력 화면에서도 이 순서대로 표시됩니다.
  • 파라미터명: SQL 내에서 사용할 파라미터 이름입니다.
  • 설명: 파라미터에 대한 설명을 입력합니다. 사용자가 입력할 때 도움말로 표시됩니다.
  • 타입: 파라미터의 데이터 타입을 설정합니다.
    • 문자열: 문자열 바인딩
    • 숫자: 숫자 바인딩
    • 텍스트: 긴 문자열용
    • SQL: SQL 코드 조각
    • 로그: 로깅용
  • 기본값: 파라미터의 기본값을 설정합니다.
  • 필수: 파라미터가 반드시 입력되어야 하는지 설정합니다. 체크하면 사용자가 값을 입력하지 않으면 SQL 실행이 차단됩니다.
  • 읽기전용: 파라미터를 읽기 전용으로 설정합니다.
  • 숨김: 파라미터를 숨김 처리합니다.
  • 비활성화: 파라미터를 비활성화합니다.

7.3 단축키 관리

SQL 실행 결과 화면에서 키보드 단축키를 눌러 다른 템플릿을 빠르게 실행할 수 있습니다.

  • 단축키: 키보드 단축키를 설정합니다. F1~F12 키 중에서 선택하여 빠른 SQL 실행이 가능합니다.
  • 단축키명: 단축키에 대한 설명적인 이름을 입력합니다.
  • 대상 템플릿: 단축키를 눌렀을 때 실행할 SQL 템플릿을 선택합니다.
  • 설명: 단축키에 대한 상세한 설명을 입력합니다.
  • 소스 컬럼: 단축키 실행 시 파라미터로 전달할 컬럼의 인덱스를 설정합니다. 1,2,3 형태로 여러 컬럼을 지정할 수 있으며, 상수값은 작은따옴표로 감싸서 1,'상수',3 형태로 사용할 수 있습니다.
  • 자동실행: 단축키를 자동으로 실행할지 설정합니다. 체크하면 조건이 만족될 때 자동으로 SQL이 실행됩니다.
  • 상태: 단축키의 활성화 상태를 설정합니다. 활성으로 설정하면 단축키가 사용 가능하며, 비활성으로 설정하면 사용할 수 없습니다.

7.4 SQL 내용 관리

  • 여러 연결에 대해 다른 SQL 내용 설정 가능
  • 연결별 SQL 탭으로 관리
  • 기본 SQL 설정 가능

8. 카테고리 할당

  • 체크박스로 템플릿을 여러 카테고리에 할당 가능

9. 템플릿 삭제

  • 템플릿의 삭제 버튼 클릭
  • 확인 메시지 후 삭제
  • 소프트 삭제 (상태가 삭제됨으로 변경)
